SECTION 1 — TICKER REPLACEMENT
Remove the scrolling ticker bar entirely. In the space it occupied between the hero section and the channel grid, replace it with a single slim social proof bar. It should be a static, full-width strip with a subtle border top and bottom. Inside it, display three short stat-style claims in a horizontal row, centered, separated by a vertical divider. Use something like: "Script approval before every video goes live" · "72-hour turnaround on short video ads" · "Full campaign report after every order." Style it minimal — small font, muted color, no icons. It should feel like a quiet confidence statement, not a feature list.

SECTION 2 — HOMEPAGE PIN PRICE FIX
The Homepage Pin add-on displayed on each channel card currently shows +$150. Change this to +$650 everywhere it appears on the channel cards across the entire page.

SECTION 3 — EMPTY CART MESSAGE Z-INDEX BUG
The "Add at least one channel to continue" message that appears when clicking the cart button while empty is currently rendering behind the cart button itself. Fix the z-index so this message always renders above the cart button and all other page elements. It should be fully readable and not obscured by anything.

SECTION 4 — DUPLICATE PR LISTING BUG
The PR Listing & Press Coverage add-on is currently appearing twice in the checkout — once added from the upsell modal before checkout, and again as a separate toggle inside the checkout panel itself. These are the same product and should be treated as a single item. The logic should work as follows: if the user already added PR Listing from the upsell modal, the toggle inside the checkout panel should reflect that as already enabled, not add a second instance. Adding it from either location should result in exactly one line item in the order summary and one charge. Remove any duplication in both the order summary lines and the grand total calculation.

SECTION 5 — PAGE SPASM AND SCROLL JUMP BUG
When any modal or panel opens — including checkout, compare, and the upsell modal — the page is jumping or spasming upward briefly before settling. This is likely caused by the body scroll lock toggling the scrollbar visibility, which shifts the page width and causes layout reflow. Fix this by adding a scrollbar-width compensation when locking body scroll — set padding-right on the body equal to the scrollbar width when scroll is locked so the layout does not shift. Also audit all modal open and close transitions for any layout-triggering CSS changes that could cause reflow and eliminate them.

SECTION 6 — PERFORMANCE OPTIMIZATION
Scan the entire codebase for the following and fix where found. Remove any redundant event listeners that are being attached multiple times, particularly on scroll and resize events. Debounce any scroll or resize handlers that are not already debounced. Check that the ticker removal does not leave behind any animation loops, setInterval calls, or requestAnimationFrame loops that are now running against nothing — cancel them. Ensure modal open and close logic is not causing unnecessary full-page re-renders. Check for any images or assets without explicit width and height attributes and add them to prevent layout shift. If any third-party scripts are loading synchronously in the head, move them to deferred loading.

SECTION 7 — FRONTEND SECURITY HARDENING
Since this is a fully frontend application with no database, the attack surface is limited but still worth hardening. Apply the following. Add a Content Security Policy meta tag in the head that whitelists only the domains actually used by the site — no wildcards. Ensure any user-entered text, specifically the project name field in checkout, is sanitised before being inserted into the DOM — use textContent rather than innerHTML wherever user input is rendered. Ensure the copy-to-clipboard function cannot be manipulated to inject unexpected content into the clipboard. If any external scripts or fonts are loaded from CDNs, add integrity and crossorigin attributes to those script and link tags. These are precautionary measures appropriate for a client-facing tool handling commercial inquiries.
